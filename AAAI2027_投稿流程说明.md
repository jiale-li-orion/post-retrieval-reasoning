# AAAI-27 投稿流程说明

本文档按 AAAI-27 官方页面和 Author Kit 整理，面向第一次投稿。重点是主会 Main Technical Track；其他 track 可能有不同 deadline 或材料要求。

## 1. 你现在要盯住的官方入口

- AAAI-27 官网：<https://aaai.org/conference/aaai/aaai-27/>
- Main Technical Track CFP：<https://aaai.org/conference/aaai/aaai-27/main-technical-track-call/>
- AAAI-27 Author Kit：<https://aaai.org/authorkit27/>

## 3. 页面限制和材料

主会投稿通常是：

- Main paper：最多 7 页 technical content。
- References：可额外占页，只能放 references。
- Supplementary material：可以提交，但 reviewer 不保证会看。关键证据、关键算法、关键实验设置不要只放 supplement。
- Supplement 截止比 full paper 晚 3 天。AAAI-27 明确列出三类 supplement：technical appendix、multimedia、code and data。
- Reproducibility checklist：所有作者都要完成。

对你的这篇论文，最稳的策略是：核心 claim、方法定义、主要结果、关键 ablation/failure decomposition 必须在 7 页内闭环；appendix 用来放额外公式、更多 case、完整 prompt、更多表格和实现细节。

## 4. 匿名投稿要求

AAAI 是匿名评审。匿名提交阶段要注意：

- 不写作者姓名和单位；LaTeX 中使用 `\usepackage[submission]{aaai2027}`。
- PDF metadata 要清理，避免作者名、机构、用户名泄露。
- 可能暴露身份的自引要匿名化，尤其是 “our previous work” 这类写法。
- 不放 acknowledgments、grant、个人主页、GitHub 私人仓库链接等会暴露身份的信息。
- 公开代码或数据如果会暴露身份，至少要使用匿名仓库或按官方允许方式处理。

实操上，初稿阶段可以正常分文件写；最终提交前再做一次匿名审计：PDF metadata、文件名、图表路径、代码链接、acknowledgment、自引措辞、supplement 里的路径和用户名。

## 5. LaTeX 模板和源码提交

AAAI-27 必须用 `aaai2027.sty` 和 `aaai2027.bst`。2027 模板相对旧版有一个重要变化：模板会自动加载 `newtxtext`、`helvet`、`courier`，论文 preamble 里不要再手动 `\usepackage{times}`、`\usepackage{helvet}`、`\usepackage{courier}`。

官方 Author Kit 还强调：

- 不要修改 `.sty` 文件。
- 不要用会改变页边距、字号、行距、caption 间距、float 间距的命令或包。
- 不要用 `hyperref` 或 `navigator`。
- 不要有页眉、页脚、页码。
- 图和字体必须能正确嵌入，不能有 Type 3 fonts。
- 最终源文件提交通常要求单个 `.tex` 文件，不能依赖多个 `\input{}` 分文件；`.bib` 可以单独提交。

当前仓库可以继续用 `sections/*.tex` 分文件写作，因为这样方便编辑。临近提交时需要增加一个 flatten 步骤，把 `main.tex` 和各 section 合并成单个提交用 `.tex`。

## 6. 审稿流程是什么样

AAAI-27 Main Track 是 two-phase reviewing：

1. Phase 1：每篇至少两个人类 review，另有一个 AI-generated supplementary review。AI review 是非决定性的，不给 formal decision。
2. Phase 1 后：SPC 在 AC 和 Program Chairs 框架下根据 review 和人工评分决定是否进入 Phase 2。Phase 1 被拒的论文会看到 reviews，包括 AI-generated review。
3. Phase 2：未被 Phase 1 拒掉的论文会获得更多 human reviews。
4. Author response：只有进入 Phase 2 的论文才有 rebuttal/feedback 机会。作者能看到所有 reviews，包括 AI-generated review，也可以回应它。
5. Discussion 和 metareview：SPC 根据 reviews、作者回应和 reviewer discussion 给出推荐，之后形成最终决定。

AAAI 明确说：人类 reviewer 不会被 AI reviewer 替代，最终决定也不是 AI 做的。AI-generated review 和 discussion summary 是辅助材料。

## 7. Reviewer 会看什么

AAAI-27 CFP 明确列出的核心标准包括：

- contribution 是否显著、是否新。
- research problem / question、method、experiment、analysis 是否有实质贡献。
- 理论或实证 claim 是否 sound。
- 是否和 AAAI 社区相关。
- 写作是否清楚。
- 是否符合 responsible research practice。
- 是否有足够 reproducibility：proof、实验细节、数据、代码、设置说明。

CFP 还特别说，AAAI 偏好能探索新问题、新方向、提出有广泛兴趣的方法或研究问题的 solid technical papers；不只是狭窄子领域里的小幅 SOTA 增量。

对你这篇，审稿人最可能追问的是：

- 这是不是一个真正的新 abstraction，而不是 memory retrieval 的重新命名？
- typed access / primitive trace / evidence view 是否有清晰定义和可操作性？
- 实验是否真的能把 access failure、view failure、decoder-use failure 分开？
- ATM-Bench 和 STALE 是否足以支持 paper 的 general claim？
- 你的方法边界是否清楚：不是通用 memory architecture，而是 runtime access / evidence view construction。

## 8. Author response 怎么准备

Author response 不是重写论文，而是短窗口内纠正误解、补关键证据、承认边界。第一次投时容易犯的错误是把 rebuttal 写成情绪化解释，或者试图回答所有细枝末节。

建议提前准备：

- 一个 claims-to-evidence 表：每个主 claim 对应哪个实验、表、图、消融。
- 一个 limitation 表：哪些问题你不解决，为什么这不是论文缺陷。
- 一个 likely objections 表：比如 “只是 top-k retrieval?”、“为什么不是 memory architecture?”、“为什么这两个 benchmark 足够？”。
- 实验脚本和结果缓存：rebuttal 期间如果需要补一张小表，必须能快速复现。

## 9. AI 工具使用政策

AAAI-27 允许作者审慎使用 generative AI 工具辅助写作，但作者对全部提交内容负责，且必须遵守 AAAI ethics 和 publication policies。

Publication Policies 还强调：

- AI system 不能作为作者。
- AI system 不能作为可引用来源。
- 如果 AI 在论文开发中起作用，需要在 manuscript 中适当记录其角色。
- 作者对文本、图、引用、附录都负全责。

实操建议：不要引用 ChatGPT/Codex 作为来源；不要让 AI 编造文献；所有文献都要人工核查。是否需要在论文里声明 AI 使用，要看最终 AAAI-27 submission form 和 Author Kit 的具体要求。

## 11. 第一次投稿的工作节奏

建议按这个顺序推进：

1. 先把 paper story 固定：问题、缺口、方法边界、实验能证明什么。
2. 同步跑主实验和写实验 section 草稿；不要等所有结果出来才写。
3. 每个实验只服务一个 claim，避免堆 benchmark。
4. Introduction 和 abstract 等实验主结果稳定后再最后收紧。
5. 截稿前一周进入 submission engineering：页数、引用、图表、匿名、checklist、supplement、代码包。
6. 截稿前至少 24 小时完成一次从 clean directory 编译，确认没有本地路径依赖。
